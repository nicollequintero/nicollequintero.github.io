I'm a Hacker Now

August 10, 2014

That was my first thought when I learned about SQL injection - "wow, I can hack into websites now". Disclaimer: I don't actually intend to do this and would probably be hard pressed to find a worthwhile site vulnerable to this hacking... But it is rewarding to further my coding knowledge and understand how vulnerabilities arise.

In essense, SQL injection is the ability to manipulate backend SQL queries in a way that causes unintended access to the data at hand. The most popular example of this is entering characters into the username or password field of a website that, once implemented into the backend query, makes the query evaluate to true without really having the proper username/password combination. For example:

user_name = " or ""="
password = " or ""="

sql = "SELECT * FROM Users WHERE Name ='" + user_name + "' AND Pass ='" + password + "'"

#once the variables are injected, the SQL statement becomes
SELECT * FROM Users WHERE Name ="" or ""="" AND Pass ="" or ""=""

Because the latter part of the OR statements (space equals space) will evaluate to true, the select statement will return all data from the Users table. Of course, getting the characters right means fiddling around to guess at how the SQL backend statement is written exactly, but it takes little exploring for a seasoned hacker to arrive at an intelligent guess.

Some strategies to combat the potential side effects of SQL injection are:

  • Limit the characters that can be entered into fields like username and password. For example, not allowing quotes or equals signs or certain strings.
  • Sanitizing your data, meaning that if you don't limit things like single quotes because you want to allow Mr. O'Malley to enter his name properly, you can at least clean the data up afterwards. Insert escape characters before quotes, for example, which will break up the syntax and not result in SQL injection.
  • Don't use dynamic SQL, use prepared statements instead. Prepared statements are basically templates for your SQL statements so that the user input isn't being inserted directly into your query.

Be safe out there, folks!
